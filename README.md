# puzzles_in_ossetian
# Веб-приложение "Ребусы на осетинском".
Автор проекта - Чувенкова Анастасия.
Введение.
Мой проект помогает сохранить и популяризировать осетинский язык, 
развивает логическое мышление, расширяет словарный запас, 
предоставляет возможность для развлечения.
Функции игры:
 1.Отображение ребусов, возможность взять подсказки(3), ввод, 
отображение правильности ответа, переключение на следующий 
ребус, возможность выйти на главную страницу.
 2.Правила игры.
 3.Возможность играть без регистрации.
 4.Регистрация по логину и паролю или через соцсети.
 5.Сохранение прогресса у зарегистрированных пользователей.
 6.Отображение лучших игроков в рейтинге.
 7.Дополнительные задания, мобильная адаптация, озвучка 
подсказок, режим соревнования, настройки игры, визуальные 
эффекты.
Чтобы запустить игру, нужно скачать Pycharm, установить библиотеки: 
PyQt5, sqlite3. Нужны файлы с ребусами: rebus1.png, rebus2.png, 
rebus3.png, rebus4.png, rebus5.png.
Руководство пользователя.
Сначала появляется стартовое окно. Нажмите на “Правила игры”, чтобы узнать как 
играть.
Нажимая “Играть”, появляется ребус.
Для удобства на экране есть буквы.
Если сложно отгадать, можно взять подсказку(всего их 3).
Чтобы проверить ответ, нужно нажать на кнопку “проверить”.
Чтобы переключиться на следующий ребус, нажмите “Следующий 
ребус”.
Также вы можете зарегистрироваться или войти для сохранения 
прогресса и отображения в рейтинге. Его можно посмотреть, нажав на 
кнопку “Рейтинг игроков”.
При верном решении всех ребусов, появляется поздравление.
Руководство программиста.
1. Модули и библиотеки:
 sys: Используется для работы с аргументами командной строки 
и выхода из приложения.
 sqlite3: Используется для работы с базой данных SQLite для 
хранения рейтинга игроков.
 PyQt5: Используется для создания графического интерфейса 
пользователя (GUI).
QApplication: Основной класс для управления 
приложением.
o QWidget: Базовый класс для всех элементов управления.
o QVBoxLayout, QHBoxLayout, QGridLayout: Макеты для 
размещения элементов управления.
o QPushButton: Кнопки для взаимодействия с 
пользователем.
o QLineEdit: Поля ввода для ввода текста.
o QLabel: Метки для отображения текста и изображений.
o QMessageBox: Всплывающие окна для отображения 
сообщений.
o QTableWidget: Таблица для отображения рейтинга игроков.
o QStackedWidget: Виджет для отображения нескольких 
страниц.
o QSizePolicy: Политика размера для элементов управления.
o QMenuBar, QAction: Меню и действия для меню.
o QPixmap: Класс для работы с изображениями.
o QFont, QMovie: Классы для работы со шрифтами и 
анимацией.
o Qt, QTimer: Основные классы для работы с событиями и 
таймерами.
o pyqtSignal: Сигналы для взаимодействия между классами.
2. Классы:
 StartWindow: Главное окно приложения с кнопками для начала 
игры, регистрации/входа, просмотра правил и рейтинга.
 LoginOrRegisterWindow: Окно для выбора между входом и 
регистрацией.
 LoginWindow: Окно для входа в систему.
 RegistrationWindow: Окно для регистрации нового 
пользователя.
 RatingWindow: Окно для отображения рейтинга игроков.
 GameWindow: Окно игры, где пользователь решает ребусы.
3. База данных:
 SQLite: Используется для хранения рейтинга игроков.
o Таблица users: Содержит следующие поля:
 name: Имя пользователя.
 score: Рейтинг пользователя.
4. Функции:
   connect_db: Подключение к базе данных SQLite и создание 
таблицы users, если она не существует.
 load_ratings: Загрузка рейтинга игроков из базы данных в 
словарь user_ratings.
 save_ratings: Сохранение рейтинга игроков из словаря 
user_ratings в базу данных.
5. Использование базы данных:
 При запуске приложения: Вызывается функция 
load_ratings() для загрузки рейтинга игроков из базы данных.
 При регистрации нового пользователя: Вызывается функция 
save_ratings() для сохранения нового пользователя в базу 
данных.
 При решении ребуса: Вызывается функция save_ratings()
для обновления рейтинга пользователя в базе данных.
6. Описание работы приложения:
1. Запуск приложения:
o При запуске приложения загружается рейтинг игроков из 
базы данных.
o Открывается главное окно (StartWindow), где 
пользователь может выбрать между игрой, 
регистрацией/входом, просмотром правил и рейтинга.
2. Регистрация/Вход:
o При нажатии на кнопку "Регистрация или вход" открывается 
окно (LoginOrRegisterWindow), где пользователь может 
выбрать между входом и регистрацией.
o При выборе "Войти" открывается окно (LoginWindow), где 
пользователь вводит имя и пароль.
o При выборе "Зарегистрироваться" открывается окно 
(RegistrationWindow), где пользователь вводит имя и 
пароль для регистрации.
3. Игра:
o После успешной регистрации или входа открывается окно 
(GameWindow), где пользователь решает ребусы.
o За каждый правильный ответ пользователь получает 20 
очков, которые сохраняются в базе данных.
o Пользователь может использовать подсказки, переходить к 
следующему ребусу и выходить из игры.# puzzles_in_ossetian
