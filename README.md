# Веб-приложение "Ребусы на осетинском"

## 1. Автор проекта
Чувенкова Анастасия

---

## 2. Аннотация (краткое описание назначения)
Проект представляет собой веб-приложение, которое помогает сохранить и популяризировать осетинский язык через решение ребусов. Приложение развивает логическое мышление, расширяет словарный запас и предоставляет возможность для развлечения. Реализовано на базе Flask, что делает его доступным для пользователей через браузер.

---

## 3. Техническое задание (перечисление функций)

### **Основные функции:**
1. Отображение ребусов с возможностью их решения.
2. Возможность использования подсказок (максимум 3 на ребус).
3. Ввод ответа и проверка его правильности.
4. Переключение на следующий ребус.
5. Выход на главную страницу.
6. Просмотр правил игры.
7. Возможность играть без регистрации (для гостей).
8. Регистрация и вход пользователей.
9. Сохранение прогресса зарегистрированных пользователей.
10. Отображение рейтинга игроков.
11. Предложение зарегистрироваться после решения всех ребусов.
12. Поздравление при решении всех ребусов.

## 5. Руководство пользователя (куда нажимать и что получится)

### **Последовательность действий:**
1. **Стартовая страница**:
   - При открытии сайта появляется главное меню с кнопками:
     - "Правила игры" – узнать правила.
     - "Играть" – начать игру.
     - "Войти/Зарегистрироваться" – войти или создать аккаунт.
     - "Рейтинг игроков" – посмотреть таблицу лидеров.

2. **Игра**:
   - Нажмите "Играть", чтобы увидеть первый ребус.
   - Для удобства на экране есть буквы осетинского алфавита.
   - Если сложно отгадать, используйте подсказки (максимум 3).
   - Введите ответ в поле и нажмите "Проверить".
   - Если ответ правильный, вы перейдете к следующему ребусу.

3. **Регистрация/Вход**:
   - После решения всех ребусов появится предложение зарегистрироваться.
   - Зарегистрируйтесь или войдите, чтобы сохранить прогресс.

4. **Рейтинг игроков**:
   - Нажмите кнопку "Рейтинг игроков", чтобы увидеть таблицу лидеров.

5. **Поздравление**:
   - Если вы решили все ребусы, появится поздравление.

## 6. Руководство программиста

### **Модули и библиотеки:**
- **Flask**: Используется для создания веб-приложения.
- **SQLite**: Хранение данных пользователей и рейтинга.
- **HTML/CSS/JavaScript**: Фронтенд приложения.
- **Python**: Бэкенд приложения.

### **Классы и маршруты:**
- **StartWindow (Главная страница)**:
  - Маршрут: `/`
  - Описание: Главное меню с кнопками для начала игры, регистрации/входа, просмотра правил и рейтинга.

- **LoginOrRegisterWindow (Выбор между входом и регистрацией)**:
  - Маршрут: `/login`, `/register`
  - Описание: Страницы для входа и регистрации пользователей.

- **RatingWindow (Рейтинг игроков)**:
  - Маршрут: `/rating`
  - Описание: Отображение таблицы лидеров.

- **GameWindow (Игра)**:
  - Маршрут: `/game`
  - Описание: Страница с ребусами, где пользователь решает задачи.

### **База данных:**
- **SQLite**:
  - Таблица `users`:
    - `id`: Уникальный идентификатор пользователя.
    - `name`: Имя пользователя.
    - `password`: Пароль пользователя.
    - `score`: Рейтинг пользователя.
    - `solved_rebuses`: Список решенных ребусов через запятую.

  - Таблица `riddles`:
    - `id`: Уникальный идентификатор ребуса.
    - `image`: Путь к изображению ребуса.
    - `answer`: Ответ на ребус.
    - `hints`: Подсказки через запятую.

### **Файлы и их использование:**
- **`app.py`**: Основной файл приложения.
- **`data/_all_models.py`**: Определение моделей `User` и `Riddle`.
- **`templates/`**: HTML-шаблоны для фронтенда.
- **`static/`**: Статические файлы (изображения, CSS, JavaScript).

### **Диаграмма базы данных:**

```plaintext
+-------------------+        +-------------------+
|      users        |        |      riddles      |
+-------------------+        +-------------------+
| id (PK)           |        | id (PK)           |
| name (UNIQUE)     |        | image             |
| password          |        | answer            |
| score             |        | hints             |
| solved_rebuses    |        +-------------------+
+-------------------+
```
